Brief history of Java

Why do we need to talk about Java first?

While Java and Kotlin are two separate languages, they are closely related. They both run on the Java Virtual Machine (JVM), and they are interoperable. In fact, Kotlin compiles to Java bytecode, which means that it can run on any platform that supports Java. Understanding some basic principles in Java will help you understand Kotlin better.

Java has two main components ...

Java Runtime Environment (JRE)


The JRE provides the libraries, the Java Virtual Machine (JVM), and other components needed to run applications written in Java. It does not include developer tools such as compilers and debuggers.

As a user, you would use the JRE to run Java programs on your system.


Java Development Kit (JDK)


The JDK includes the JRE as well as a set of development tools for writing and running Java programs. These tools include the Java compiler (javac), an archiver (jar), a documentation generator (javadoc), and other tools needed in Java development.

As a programmer, you would use the JDK to develop Java applications.

Introduction to Kotlin

What is Kotlin?

It was developed by JetBrains and officially released in 2016.

Kotlin is a statically-typed programming language that runs on the Java Virtual Machine (JVM).

Kotlin is fully interoperable with Java, which means that you can use Java libraries in Kotlin and vice versa.

Kotlin was designed to improve on Java's shortcomings, and it is considered a modern alternative to Java.

Some of the key features of Kotlin include:

  • Null safety
  • Extension functions
  • Smart casts
  • Coroutines
  • Operator overloading
  • Companion objects
  • Data classes

It has modern and intuitive syntax, and it is designed to be concise and expressive.

Object-oriented programming

Object-oriented programming

There are four main principles in OOP ...

  1. Encapsulation
    is a concept of controlling access to the internal state of an object, protecting it from unauthorized access and ensuring data integrity.
  2. Inheritance
    enables class to have the same behavior as another class by inheriting its properties and methods.
  3. Polymorphism
    allows us to define one interface or method that can have multiple implementations. It means that the same method or property could exhibit different behavior in different instances of object implementing given interface.
  4. Abstraction
    is a mechanism to represent the object features without exposing the actual implementation details. In other words, user of such object only needs to know what it does, not how it does it.

Kotlin Data Types

There are two groups of data types in Java and therefore in Kotlin...

Primitive types

  • Numbers, characters, and boolean values ...
  • In Java, there primitive types are the basic data types that are built into the language.
  • Kotlin has representation corresponding to Java primitive types, but unlike Java, they are all objects.
  • Because they are objects and as such they have methods and properties.
  • When Kotlin code gets compiled, the Kotlin compiler will convert these objects to Java primitive types.

Non-primitive types

  • String, Arrays and Classes, ...
  • Variables that are assigned a non-primitive type are called reference variables.
  • We will talk more about reference variables later.

All Kotlin types are not nullable by default, which means that they cannot be assigned a null value. To allow a variable to be null, you need to declare it as nullable, which we will talk about later.

The two's complement

In Java, two's complement is used for the representation of signed integers. A negative number is represented as the two's complement of its absolute value. This allows positive and negative numbers to be added together directly.

  • For positive integers, the two's complement representation is the same as their binary representation.
  • For negative integers, you use binary representation of the absolute value of the number, then you invert each bit, and finally add 1 to the least significant bit (LSB).
Example: Binary representation of -5
  1. The binary representation of absolute value of -5 (=5) is 0101
  2. Inverting each bit gives you 1010
  3. Adding 1 to the least significant bit gives you the binary representation of -5, which is: 1011

Type checks

Any type

Any is the root of the Kotlin class hierarchy. It is a super type of all non-nullable types.

Similarly, Any? is a super type of all nullable types.

This means anytime we don't know the type of a variable, parameter or return type, we can use Any to accept any type.

Any type is equivalent to Java's Object type.

Type Casting

Type casting is a method of converting one data type to another.
Keep in ming, that type casting may result in data loss!

Implicit Casting

This is also known as automatic type conversion. This is done by the JVM when we assign a value of smaller data type to a variable of a bigger data type. 

						
								val number: Int = 100
								val bigNumber: Long = number     		// Implicit type casting, no data loss
								val floatNumber: Float = bigNumber		// Implicit type casting, no data loss

								println(floatNumber)	// 100.0

Explicit Casting

Also known as manual type conversion, we perform casting explicitly.
Upcasting = assigning smaller data type to larger data type - generally doesn't result in data loss.
Down casting = assigning larger data type to smaller data type - may result in data loss. 

						
							val preciseNumber: Double = 100.04
							val wholeNumber = preciseNumber as Int	// Explicit type casting, resulting in data loss (precision)

							System.out.println(wholeNumber)		// 100

null

In Java/Kotlin, null is a special value that represents the absence of an instance. It is used to indicate that a reference variable doesn't point to any memory location or object.

  1. Reference Variables:
    In Java, you can assign null to any reference variable (non-primitive types: objects, array, interface, etc).
  2. Default Value:
    For object reference types, the default value is null when they are defined as class members and not explicitly initialized.
  3. NullPointerException:
    If you try to invoke a method or access a property on a reference variable with a null value, you will get a NullPointerException. This is a runtime exception in Java.
  4. Comparison:
    You can use null in comparison operations. For example, to check if an object is null, you can use if (object == null).
  5. Assigning Null:
    Assigning null to a variable makes it eligible for garbage collection if there are no other references to the object.
  6. Method Argument:
    null can be passed as an argument to a method and can also be returned from a method.

Bitwise Operators

& binary and sets bit to 1 if both bits at the same positions are 1 
									
										val a = 5 // binary: 0101
										val b = 3 // binary: 0011

										val a = 5 // binary: 0101

| binary xor sets bit to 1 if of bits at the same positions are different 
									
										val a = 5 // binary: 0101
										val b = 3 // binary: 0011

										val result = a ^ b // binary: 0110, decimal: 6

										val a = 5 // binary: 0101

										val result = ~a // binary: 1010, decimal: -6
<< signed left shift moves all bits by a given number of bits to the left 
									
										val a = 5 // binary: 0101

										val result = a << 1 // binary: 1010, decimal: 10
>> signed right shift moves all bits by a given number of bits to the right 
									
										val a = 5 // binary: 0101

										val result = a >> 2 // binary: 0001, decimal: 1
>>> unsigned right shift same as the signed right shift, but the vacant leftmost position is filled with 0 instead of the sign bit 
								
									val a = -5 // binary equivalent of -5: 11111111111111111111111111111011

Arrays

Arrays are used to store multiple values of the same type in one variable. You can declare arrays of both primitive and non-primitive types.

Arrays have a fixed size, which must be set during declaration. 

					
						String[] languages = new String[4] // will create array of 4 elements

You can change value of an array by its index (zero-based). If you try to assign value outside of array bound, you will get an error.

					
						String[] languages = new String[4]
						languages[0] = "Java"
						languages[1] = "Python"
						languages[2] = "C++"
						languages[3] = "JavaScript"

						// this will cause a runtime exception: Index 4 out of bounds for length 4
						languages[4] = "Kotlin"

You can also declare an initialize array in one step

					
						String[] languages = new String[] {"Java", "Python", "C++", "JavaScript"}

						// or just
						String[] languages = {"Java", "Python", "C++", "JavaScript"}

						String clothes = null

						if (temperature > 20) {
							clothes = "T-Shirt"
						} else {
							clothes = "Sweater"
						}
You can write the same thing in shortened form 
					
						if (temperature > 20) clothes = "T-Shirt" clothes = "Sweater"
Or it can be written using ternary operator like so: 
					
						String clothes = temperature > 20 ? "T-Shirt" : "Sweater"

switch

The switch operator is generally used in place of multiple if ... else if ... else statements. It provides better code readability. 

						
							switch (direction) {
								case 1 -> System.out.println("North")
								case 2 -> System.out.println("East")
								case 3 -> System.out.println("South")
								case 4 -> System.out.println("West")
							}
You can also return a value from switch 
						
							String value = switch (direction) {
								case 1 -> "North"
								case 2 -> "East"
								case 3 -> "South"
								case 4 -> "West"
								default -> "Invalid direction"
							}
Note that this is an enhanced switch syntax, which was introduced in Java 13.

switch

Prior to Java 13, switch statement would be written like this: 

						
							switch (direction) {
								case 1:
									System.out.println("North")
									break
								case 2:
									System.out.println("East")
									break
								case 3:
									System.out.println("South")
									break
								case 4:
									System.out.println("West")
									break
							}
You exit switch by break statement, but you can also do so by return.
The default keyword specifies a branch to run when there is no case match. 
						
							switch (direction) {
								case 1:
									return "North"
								case 2:
									return "East"
								case 3:
									return "South"
								case 4:
									return "West"
								default:
									return null
							}

While Loop / Do While Loop

The while evaluates condition at the beginning of the loop block, before any code is executed.

					
						while (condition) {
						  // code block to be executed
						}

The do while first executes code block once, and evaluates condition the condition.

					
						do {
						  // code block to be executed
						} while (condition)

For Loop

For loop executes a block for a given number of iterations. The evaluation of for loop is decided into 3 statements:

  • init statement is executed once before the execution of the code block
  • condition statement defines the condition for executing the code block
  • after statement is executed every time after the code block has been executed
						
							for (init condition after) {
							  // code to be executed
							}

This would be an example of executing code in a loop 10 times, where the loop is controlled by variable i with initial value of 0, incremented by 1 each time the loop is executed until the condition of i < 10 is no longer true. 

						
							for (int i = 0 i < 10 i++) {
							  // code to be executed
							}

You can also nest loops.

						
							for (int i = 0 i < 10 i++) {
								for (int j = 0 j < 3 j++) {
									// code to be executed
								}
							}

For Loop

								for (int i = 1 i < 10 i = i * 2) {

									System.out.println(i)

								}
4 times. Values printed will be 1, 2, 4, and 8.

For Each Loop

If you have an array, can iterate it using for loop.

						
							String[] languages = {"Java", "Python", "C++", "JavaScript"}

							for (int i = 0 i < languages.length i++) {

								System.out.println(languages[i])

							}

You can also use more convenient for each loop.

						
							String[] languages = {"Java", "Python", "C++", "JavaScript"}

							for (String language : languages) {

								System.out.println(language)

							}

Break / Continue

The break statement is used to end the loop immediately.

						
							for (int i = 0 i < x i++) {

								if (breakCondition) {
									break
								}

							}
							// if breakCondition was met, the loop will end

The continue statement is end current iteration, but the loop will continue with next iteration.

						
							for (int i = 0 i < x i++) {

								if (continueCondition) {
									continue
								}

								// if continueCondition was met, this code will not be executed
							}
